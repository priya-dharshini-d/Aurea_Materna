# Aurea Materna – System README

## 📁 Project Structure

```
Aurea_Materna/
├── esp32/
│   └── MaternalMonitor.ino      ← Upload this to ESP32
└── ml_backend/
    ├── app.py                   ← Flask server (main entry)
    ├── model.py                 ← ML predictor + feature extraction
    ├── train_model.py           ← (Optional) train RandomForest model
    ├── risk_model.pkl           ← Generated by train_model.py
    ├── requirements.txt         ← Python dependencies
    └── dashboard.html           ← Open in browser
```

---

## 🔧 Step 1 – Upload ESP32 Code

1. Open **Arduino IDE**
2. Install libraries (Sketch → Include Library → Manage):
   - `Adafruit SSD1306`
   - `Adafruit GFX Library`
   - `DallasTemperature`
   - `OneWire`
   - `HTTPClient` (usually built-in for ESP32)
3. Open `esp32/MaternalMonitor.ino`
4. Set your `WIFI_SSID`, `WIFI_PASS`, and `SERVER_IP` at the top of the file.
5. Select **Board**: `ESP32 Dev Module`
6. Select your **COM port**
7. Upload ✓

The ESP32 will connect to WiFi and POST JSON data to the backend.

---

## 🐍 Step 2 – Set Up Python Backend

```bash
cd ml_backend

# Create & activate virtual environment (recommended)
python -m venv venv
venv\Scripts\activate        # Windows
# source venv/bin/activate   # Linux / Mac

# Install dependencies
pip install -r requirements.txt

# (Optional but recommended) Train the ML model first
python train_model.py
```

---

## 🚀 Step 3 – Start the Flask Server

```bash
# Make sure you are inside ml_backend/ and venv is active
python app.py
```

- The server runs on `http://0.0.0.0:8000`.
- If no data is received from ESP32 for 3 seconds, it switches to **Demo Mode** (simulated data).

---

## 🌐 Step 4 – Open the Dashboard

Open `ml_backend/dashboard.html` in your browser.

> **Tip**: If you get CORS issues, serve it with:
> ```bash
> python -m http.server 3000
> ```
> Then visit `http://localhost:3000/dashboard.html`

## 📡 API Endpoints

| Method | URL              | Description                        |
|--------|------------------|------------------------------------|
| POST   | `/data/ingest`   | Receives data from ESP32           |
| GET    | `/`              | Health check                       |
| GET    | `/data/latest`   | Latest sensor reading + prediction |
| GET    | `/data/history?n=50` | Last N readings               |
| GET    | `/stream`        | Live Server-Sent Events (SSE) stream|

---

## 🤖 ML Risk Levels

| Level | Label         | Trigger Conditions                                      |
|-------|---------------|---------------------------------------------------------|
| 0     | Normal        | All vitals in expected range                            |
| 1     | Moderate Risk | Mild fever (38–39°C) or BPM slightly out of range      |
| 2     | High Risk     | High fever (>39°C), extreme BPM, or sudden movement    |

If `risk_model.pkl` is missing, the system uses a **rule-based fallback** automatically.
